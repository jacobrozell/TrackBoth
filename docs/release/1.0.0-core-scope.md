## Product promise (1.0.0)

TrackBoth helps people **log habits and vices daily**, **see progress over time**, and **stay motivated** — with local data they control via JSON export.

One sentence: **Daily boolean + quantity tracking for habits and vices, with goals, history, charts, motivations, and settings.**

---

## Locked core loop

These six surfaces ship in 1.0.0. **No new tabs. No new primary flows.**

| Surface | Ships | Core behavior |
|---------|-------|---------------|
| **Home** | ✅ | Daily log; habits/vices sections; week mini-calendar; streaks; milestone banners; LoggingSheet |
| **Goals** | ✅ | Boolean goals (weekly/monthly/yearly); quantity goals |
| **History** | ✅ | Calendar + list; edit entries; filters |
| **Charts** | ✅ | Line, bar, heatmap; quantity views |
| **Motivations** | ✅ | Feed + primary motivation for vices (not gamified scroll) |
| **Settings** | ✅ | Export/import JSON, delete all data, themes, week start, share app |

**Onboarding** ships as first-launch flow with habit/vice presets.

**Vice-specific core** (part of Home/History, not separate tabs): avoided/not-avoided semantics, optional slip timer, money-saved estimate, mood chips on log.

## Iteration mode (now → ship)

Engineering for core features is **largely complete**. Remaining work is **confidence**, not expansion.

### Do iterate on

1. **Correctness** — vice vs habit semantics everywhere; streak boundaries; export round-trip
2. **Partial features** — quantity goals and quantity charts (manual QA passes)
3. **Polish** — layout on device/iPad, Dynamic Type, VoiceOver on Charts
4. **Release ops** — TestFlight, dogfood, App Store assets, hosted privacy URL
5. **Bug fixes** — P0/P1 only within locked scope

### Do not iterate on (until after 1.0)

- New tabs or major navigation changes
- Widget, watch, notifications, shortcuts
- CloudKit / automatic sync
- Gamification (achievements, motivation game)
- Schema changes without updating schema baseline doc
- "Nice" refactors unrelated to ship blockers
