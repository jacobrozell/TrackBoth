## Pre-release context

| Fact | Implication |
|------|-------------|
| **No App Store users yet** | No production data to migrate. Breaking schema changes are acceptable until 1.0 ships. |
| **Goal is schema lock for 1.0.0** | Treat the tables below as the **1.0 contract**. Avoid drive-by field renames or entity splits without updating this doc. |
| **Post-1.0 changes require migration** | After first public release, every persisted field change needs `VersionedSchema`, `SchemaMigrationPlan` stages, export compatibility, and tests. |
| **CloudKit / iCloud sync deferred** | 1.0 ships **local-only SwiftData** + JSON export/import. Do not reintroduce CloudKit without revisiting the deferred migration plan. |

## SwiftData schema (locked for 1.0.0)

**Versioned schema:** `TrackBothSchemaV1` → `Schema.Version(1, 0, 0)`  
**Code:** `TrackBoth/Domain/Data/TrackBothSchema.swift`  
**Models:** `Metric`, `MetricEntry`, `Goal`  
**Migration plan:** `TrackBothMigrationPlan` — **empty stages** (no version ladder yet; correct for pre-release)

### Metric

| Field | Type | Default / notes |
|-------|------|-----------------|
| `id` | UUID | Primary key |
| `name` | String | |
| `createdAt` | Date | |
| `habitType` | HabitType | `.positive` or `.vice` |
| `primaryMotivation` | String? | Vice creation motivation |
| `hasBeenLogged` | Bool | `false` — user has ever explicitly logged |
| `costPerUnit` | String? | Decimal string for vice savings |

### MetricEntry

| Field | Type | Default / notes |
|-------|------|-----------------|
| `id` | UUID | Primary key |
| `metricID` | UUID | FK to Metric (not `@Relationship`) |
| `date` | Date | Start of calendar day |
| `value` | Bool | Habit done / vice avoided semantics |
| `motivation` | String? | Daily motivation |
| `starred` | Bool? | Primary motivation flag |
| `details` | String? | Habit detail subtitle |
| `quantity` | Int? | Quantity tracking |
| `unit` | String? | e.g. `"minutes"`, `"times"` |
| `mood` | String? | Emoji mood on log |
| `hasBeenLogged` | Bool | `false` — entry explicitly saved by user |

**Invariant:** At most one entry per `(metricID, calendar day)`.

## JSON export schema (locked for 1.0.0)

**Current version:** `4` (`TrackBothExport.currentSchemaVersion`)  
**Code:** `TrackBoth/Domain/Data/TrackBothExport.swift`

| Version | Contents |
|---------|----------|
| 1–3 | metrics + entries (legacy import supported) |
| **4** | metrics + entries + **goals** + `costPerUnit` on metrics |

Export is the user-facing portability path for 1.0 (not iCloud sync).

## Persistence stack (1.0.0)

```
TrackBothApp
  └─ BootstrapStoreRecovery.makeContainer()
       ├─ local-only persistent ModelConfiguration("TrackBoth")
       └─ in-memory fallback + MigrationRecoveryView banner
```

- **Entitlements:** App Group only (`group.com.trackboth.app`) — no iCloud/CloudKit
- **Widgets:** App Group snapshot pattern (no direct SwiftData in extension)

---

## Launch-time “migration” (dev backfill only)

`MigrationUtils.runMigrationIfNeeded` runs on every launch but targets **pre-release internal state**, not shipped-user upgrades:

1. **Cost backfill** — one-time copy from legacy `MetricCostStore` UserDefaults → `Metric.costPerUnit`
2. **Logged-status backfill** — promotes `hasBeenLogged` on entries/metrics from existing dev data

These are safe to simplify or remove **before 1.0 ships** if no longer needed. They are **not** a substitute for post-release `SchemaMigrationPlan` work.

## Rules for schema changes

### Before 1.0.0 ships (now)

- Prefer locking fields over renaming.
- If you add a field: optional + default on the `@Model` property.
- Update `DataSchemaSpec.md`, this file, and export records if the field is user-visible in export.
- No `SchemaMigrationPlan` stage required unless you introduce `TrackBothSchemaV2`.

### After 1.0.0 ships

1. Add `TrackBothSchemaV2` (or next version) — never mutate V1 in place.
2. Add `MigrationStage`(s) to `TrackBothMigrationPlan`.
3. Extend JSON export with backward-compatible import for prior versions.
4. Add unit tests for migration idempotency and export round-trip.
5. Update this document with the new version ladder.

---

## Explicitly out of scope for 1.0.0

- SwiftData + CloudKit automatic sync
- Legacy manual `TrackBothBackup` CloudKit records (removed)
- Multi-device sync UX in Settings
- Production migration runbooks for existing App Store users (none exist)
