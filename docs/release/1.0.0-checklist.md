## What still needs to be done for confident 1.0.0

### 1. Physical device QA (highest priority)

Unchecked in [`1.0.0-ship-checklist.md`](1.0.0-ship-checklist.md) and blank in [`QA-Signoff-RC1.md`](QA-Signoff-RC1.md):

- [ ] Fresh install → onboarding → add habit → log → history
- [ ] Vice logging: "avoided" vs "not avoided" displays correctly everywhere
- [ ] JSON export → import round-trip on device
- [ ] **7-day dogfood**: daily logging without crashes or data loss
- [ ] iPad landscape Home (if claiming iPad support)

**Partial features** — need explicit device passes:
- [ ] Quantity goals
- [ ] Quantity charts

### 2. Accessibility manual sign-off

Automated checks exist (`WCAGContrastTests`, identifier tests). Manual items from [`accessibility/1.0-nutrition-label-checklist.md`](../accessibility/1.0-nutrition-label-checklist.md):

- [ ] VoiceOver walkthrough (especially **Charts** summaries)
- [ ] Largest Dynamic Type on Home + History (no clipped actions)
- [ ] Reduce Motion check
- [ ] Midnight theme readability spot-check

### 3. TestFlight RC → formal QA sign-off

- [ ] Archive Release build (`TrackBoth` scheme, no widget embed)
- [ ] Upload to App Store Connect
- [ ] Run all 12 smoke flows in [`QA-Signoff-RC1.md`](QA-Signoff-RC1.md)
- [ ] Confirm **no open P0/P1** bugs
- [ ] Get explicit **Approved** decision on the sign-off doc

### 4. App Store Connect & legal

- [ ] **Host** [`privacy.html`](../privacy.html) (live URL for App Store Connect)
- [ ] Complete privacy nutrition labels + age rating questionnaires
- [ ] Export compliance (standard on-device encryption declaration)
- [ ] Support URL (required)

### 5. App Store assets

- [ ] Screenshots (6.7" iPhone required; iPad if supported)
- [ ] Verify 1024×1024 app icon
- [ ] Description, subtitle, keywords
- [ ] Use `TrackBothScreenshots` scheme for consistent demo-data shots

### 6. Release gates & tagging

- [ ] Fill sign-off table in [`1.0.0-ship-checklist.md`](1.0.0-ship-checklist.md) (Engineering / QA / Release)
- [ ] After App Store approval: tag `1.0.0` in git
- [ ] Update feature inventory to "1.0 shipped"
- [ ] Open 1.1 milestone (widget is planned first post-release feature)

## Path to ship

```
Device QA + dogfood → TestFlight RC1 → QA sign-off → Store assets + privacy URL → App Store submit → Tag 1.0.0
```

**Bottom line:** Engineering is done. To be **fully confident** in 1.0.0, focus on:

1. Real-device validation (especially export/import, vice semantics)
2. 7-day personal dogfood
3. TestFlight RC with completed QA sign-off
4. Store listing + hosted privacy policy
